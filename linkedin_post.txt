20 autonomous browser agents. Simultaneously. Each one piloting a real Chromium instance, navigating live websites, and filing UX reports like a tiny unhinged intern army. 🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖🪖

Here's what immediately exploded 💥

𝗣𝗿𝗼𝗯𝗹𝗲𝗺 𝟭: 𝗧𝗵𝗲 𝗱𝗮𝘁𝗮𝗯𝗮𝘀𝗲 𝗱𝗶𝗲𝗱 𝗶𝗻𝘀𝘁𝗮𝗻𝘁𝗹𝘆 💀

MariaDB got a 256MB memory limit and absolutely did not appreciate 20 workers asking it questions at the same time. It just. died. Fly.io sent me a very polite email about it. We bumped it to 1GB and moved on with our lives.

𝗣𝗿𝗼𝗯𝗹𝗲𝗺 𝟮: 𝟭𝟭 𝗼𝘂𝘁 𝗼𝗳 𝟮𝟬 𝘁𝗲𝘀𝘁𝘀 𝗳𝗮𝗶𝗹𝗲𝗱 😵

Shared CPU VMs. Twenty Chromium instances all trying to render screenshots at the same time on fractional CPUs. They were stepping on each other. Playwright gave up after 30 seconds on each one. The fix was upgrading to dedicated CPU machines, which costs approximately the same as a large coffee ☕ for a 3-hour run.

𝗣𝗿𝗼𝗯𝗹𝗲𝗺 𝟯: 𝗮 𝘃𝗲𝗿𝘆 𝘀𝗽𝗲𝗰𝗶𝗳𝗶𝗰 𝗮𝗻𝗱 𝗱𝗲𝗹𝗶𝗴𝗵𝘁𝗳𝘂𝗹𝗹𝘆 𝘀𝘁𝘂𝗽𝗶𝗱 𝗣𝗹𝗮𝘆𝘄𝗿𝗶𝗴𝗵𝘁 𝗴𝗼𝘁𝗰𝗵𝗮 🎭

`set_default_timeout()` does NOT cover navigation timeouts. There is a completely separate method for that. `set_default_navigation_timeout()`. They are different. They have always been different. This is documented. I did not read the documentation carefully enough. We fixed it.

𝗧𝗵𝗲 𝗿𝗲𝘀𝘂𝗹𝘁𝘀 📊

Run 2 — dedicated CPU, 60s timeout: 𝟭𝟲/𝟮𝟬 ✅
Run 3 — navigation timeout fixed: 𝟭𝟲/𝟮𝟬 ✅
